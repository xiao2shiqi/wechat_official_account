如果你也和我一样，是一个重度的 **Obsidian** 使用者，你一定经历过这种痛苦：

在 `Properties`（属性栏）里想加个链接，结果发现它只支持纯文本，或者那个丑陋的 URL 回显。更要命的是，当你改了目标文件的名字，属性栏里的链接往往就**断了**，你得手动一个一个去修。

今天凌晨，Obsidian 发布了 **1.11.5** 正式版。看到 Changelog 的那一刻，我长舒了一口气：**Properties 里的 Markdown 链接，终于被官方彻底修好了。**

这次更新虽然看似是个“小版本”，但里面的几项改进，每一项都精准踩在了老玩家的痛点上。

### 01. Properties 终于支持 Markdown 链接了

这是本次更新的核心。现在，你可以在文本和列表类型的属性（Properties）中，直接使用普通的 Markdown 链接语法（如 `[[笔记名称]]` 或 `[显示文字](链接地址)`）。

但这还不是最酷的。最让我惊喜的是：**当目标文件被移动或重命名时，这些属性栏里的链接会自动更新。**

以前，如果我们想在属性里维护笔记之间的关系，往往要借助 `Dataview` 插件或者忍受脆弱的手动维护。现在，官方原生支持了这种“动态关联”，这意味着你的笔记网络在属性层级也变得**坚不可摧**了。

### 02. Keychain：为插件安全“上锁”

你是否也曾在多个插件里反复输入同一个 `OpenAI API Key` 或者 `Google 翻译密钥`？

1.11.5 新增了一个 **Keychain (钥匙串)** 设置项。这是一个专门为插件存储“秘密”的地方。
*   **统一管理**：不再需要每个插件都存一份密钥。
*   **安全性**：利用系统的安全存储机制保护敏感信息。
*   **即将到来**：虽然相关的插件 API 还在赶来的路上，但这预示着 Obsidian 的插件生态正在走向更加专业和安全的方向。

如果你也担心密钥泄露，这个功能的出现绝对是个定心丸。

### 03. 细节控的极致：它更像“原生”软件了

Obsidian 一直被一些人抱怨“不够 Mac”，或者“长得有点 Geek”。这次更新，开发团队在审美和细节上又卷了一把：

*   **Mac 原生体验**：在 macOS 上，所有的开关（Toggles）和滑动条（Sliders）现在都经过了重新设计，完美贴合 **macOS** 的原生视觉风格。
*   **设置栏图标**：说实话，以前的设置菜单满眼全是文字，找起来挺费劲。现在每个板块都加上了直观的**小图标**，找设置快了不止一秒。
*   **Inter 变量字体**：编辑器现在采用了 `Inter Variable` 字体。相比之前的版本，字重调节更细腻，阅读体验也更丝滑。

### 04. 效率党的“解药”

还有几个能让你操作快到飞起的小改动：

*   **粘贴自动转链接**：当你选一段文字，直接按粘贴（Ctrl/Cmd + V）一个 URL，它会自动把这段文字变成一个 Markdown 链接。
*   **默认打开文件设置**：现在你可以设置启动时默认打开：最后一次打开的文件、新笔记、特定笔记，甚至是**当天的日记**。
*   **URI 弹窗控制**：对于喜欢折腾自动化（如 URL Scheme）的同学，现在可以在 `paneType` 里指定是在新标签、分屏还是**弹出窗口**里打开笔记了。

### 总结一下

Obsidian 1.11.5 并不是一个堆砌新功能的版本，它更像是一场**“还债”之旅**。

它还清了 Properties 链接管理的债，还清了插件密钥安全的债，也还清了 UI 细节上的一丝不苟。对于我们这些每天要在里面待上几个小时的人来说，这些改进带来的“丝滑感”，远比发布一个花哨的新功能要重要得多。

**建议所有用户立即更新。**

如果你在更新后发现了其他有趣的小细节，或者遇到了什么 Bug，欢迎在评论区和我聊聊。
