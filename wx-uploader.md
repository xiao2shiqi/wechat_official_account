元属性（常用）：

| key          | value                        | example                                                                                                                                                                                                                                           |
| ------------ | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| title        | "我的文章标题"                     | **（必须）** 设置文章的标题。这个标题将用于微信公众号文章的最终标题。                                                                                                                                                                                                             |
| published    | `"draft"` / `"true"` / `(空)` | **（核心功能）** 控制文章的发布状态。<ul><li>如果值为 `true` (布尔值 `true` 或字符串 `"true"` 都可以)，在批量上传目录时，该文件将被跳过。</li><li>如果值为 `"draft"`，表示文章已经作为草稿上传到了微信公众号。</li><li>如果这个字段不存在、为空，或者不是 `"true"`，那么在批量上传时，工具会处理并上传这篇文章。</li><li>上传成功后，工具会自动将此字段的值更新为 `"draft"`。</li></ul> |
| cover        | "cover.png"                  | **（可选）** 指定文章的封面图片路径。<ul><li>路径可以是相对路径（相对于当前 Markdown 文件）或绝对路径。</li><li>如果设置了此字段但找不到对应的图片文件，并且配置了 OpenAI API 密钥，工具会尝试自动生成一张新的封面并保存到指定路径。</li><li>如果此字段**未设置**，并且配置了 OpenAI API 密钥，工具会自动分析文章内容，生成一张风格化的封面图片，并自动填充此字段。</li></ul>                    |
| description  | "这篇文章是关于..."                 | **（可选）** 文章的摘要或描述。这个描述会用于两个地方：<ul><li>作为微信公众号文章的摘要。</li><li>在自动生成封面图片时，AI 会分析这段描述（以及正文内容）来创造一个更贴合的场景。</li></ul>                                                                                                                                   |
| theme        | "lapis"                      | （可选） 设置文章在微信公众号中的渲染主题样式。支持以下几种主题：<ul><li>default</li><li>lapis</li><li>maize</li><li>orangeheart</li><li>phycat</li><li>pie</li><li>purple</li><li>rainbow</li></ul>                                                                              |
| code         | "github-dark"                | （可选） 为文章中的代码块设置语法高亮的主题样式。支持以下几种高亮主题：<ul><li>github</li><li>github-dark</li><li>vscode</li><li>atom-one-light</li><li>atom-one-dark</li><li>solarized-light</li><li>solarized-dark</li><li>monokai</li><li>dracula</li><li>xcode</li></ul>         |
| **`author`** | "作者姓名"                       | **（可选）** 指定文章的作者。这个字段以及其他未在上面列出的自定义字段（如 `tags`）会被保留，但当前版本的工具在上传到微信时不会直接使用它们，它们主要用于您自己的内容管理。                                                                                                                                                       |

### 公共 Prompt

人设 Prompt （配合 Gemini Deep Research 使用）：

历史、心理咨询和自我成长、哲学类：
```sh
请按下以下要求完成深度研究：我现在要写一篇公众号，你善于使用简单的语言来帮助人们理解复杂事情的来龙去脉。在深度研究中你需要以我作为第一人称，写下对这个主题的思考，输出的内容里面，不要使用表格，不要列出参考文献，不要在文章中使用数字编号，有序和无序编号等，因为这样不利于用户在手机上阅读，为了让内容读起来更加简单，文字之间段落清晰，段落只用一级标题，例如：1,2,3,4，不要扩展到二级标题，中文、英文、数字之间要有一个空格。现在需要你深入研究的主题是：xxxxxx， 主要要表达的观点和内容是：xxxxxx
```


科学，科普类，软件，编程 Prompt（配合 Gemini Deep Research 使用），适用于 AI 话题：
```sh
请按以下要求生成深度分析报告，你是一位精通计算机科学、AI、软件工程和编程的大师，你善于使用简单的语言和形象的比喻来帮助人们通俗的理解复杂的计算机科学里面深邃难懂的理念。你也善于引用计算机行业的权威资料和名人的言论来证明自己的理论的可靠性，你使用我的第一人称和客观的视角，输出对这个主题的深入思考，不要使用表格，不要列出参考文献，正文中不要出现参考文献和参考文献的引用符号以及不要在文章中使用数字编号，因为这样不利于用户在手机上阅读，也尽可能的少用项目符号和数字符号来减少AI 和机械感，为了让内容读起来更加简单，文字之间段落清晰，段落只用一级标题，例如：1,2,3,4，不要扩展到二级标题，每个段落文字多，文字行间充满人文的表达和输出。只需要参考英文材料，不需要参考中文材料，现在需要写一篇文章发布到微信公众号里面，文章标题你可以根据公众号标题党的风格，但是这次需要深入研究的主题是：xxxxxxxx
```

政治、经济、社会和管理话题：
```sh
请按以下要求生成深度分析报告：
您是一位资深的社会和文化专家，你非常社会热点舆论话题，你善于使用简单的语言来帮忙社会热点事件背后的本质，你也善于引用和主题相关的权威资料和经典书籍。来证明自己理论的可靠性，你需要以一个观察者的视角和第三人称，写下对这个主题的深入思考，文字需要充满理性和有数字的数据分析，不要有任何个人的情感化的表达,在深入思考输出里面，但是不要使用表格，正文中不要出现参考文献的标记符号以及不要在文章中使用数字编号，因为这样不利于用户在手机上阅读，也尽可能的少用项目符号和数字符号来减少 AI 的机械感，为了让内容读起来更加简单，文字之间段落清晰，段落只用一级标题，例如：1,2,3,4，不要扩展到二级标题，并且考虑到微信平台读者的耐心，现在需要写一篇文章发布到微信公众号里面，需要你深入思考的文章主要内容是（请你按微信公众号的标题党风格自己取标题）：xxxxx
```

英文话题：完全使用 英文的信息来源做深度研究
